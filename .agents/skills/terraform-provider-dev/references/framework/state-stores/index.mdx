# State stores

<Highlight>

State store support in framework is currently experimental and offered without compatibility promises
until Terraform core support for state stores is generally available.

State store support can be enabled in Terraform v1.15+ alpha releases by running Terraform commands with
the `-enable-pluggable-state-storage-experiment` flag or setting the environment variable `TF_ENABLE_PLUGGABLE_STATE_STORAGE`
to any value.

</Highlight>

State stores are an abstraction that allow Terraform to persist its [state](/terraform/language/state) data files in remote objects.
State stores replace the existing [backend](/terraform/language/backend) concept, which are built directly into Terraform, with
provider-defined logic.
